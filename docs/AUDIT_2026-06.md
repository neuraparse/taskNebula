# TaskNebula — Master Gap Report (Audit 2026-06)

**Date:** 2026-06-12 · **Repo state:** v0.3.4 (commit c519e8e)
**Method:** 28 parallel domain auditors + 3 market researchers produced ~330 raw findings, deduplicated to ~90 canonical gaps, then adversarially verified by an independent critic pass against the same commit (Appendix A). Refutations from the critic pass have been applied to the body below — notably, the original "main is currently red (openapi drift test failing)" claim was removed after the critic reproduced 5/5 passing openapi tests at c519e8e.

---

## 1. Executive Summary

28 auditors produced ~330 raw findings; after deduplication they consolidate to ~90 canonical gaps. Five themes dominate:

1. **Tenant isolation is the #1 risk.** The Postgres RLS that CLAUDE.md, `.claude/rules/database.md`, and `docs/ARCHITECTURE.md` all claim is mandatory **does not exist** — zero `CREATE POLICY` statements across all 54 migrations (confirmed independently by 7 auditors). Isolation rests entirely on per-route `WHERE` clauses, and **~20 API surfaces skip even those**, allowing cross-org reads _and writes_ (workflows, sprint issues, permission schemes, project members, issue links/activities, hybrid search, the SSE event bus, analytics, watchers, saved filters, and more).
2. **Backends built, last mile broken.** Large subsystems are genuinely implemented but unreachable or non-functional at the seam: Cmd+K search 405s on every query, the `/api/search` JQL route references a non-existent column, the embedding worker has zero callers (all vector features inert), the AI Sidecar is a hardcoded stub over a complete `/api/ask` RAG endpoint, API keys are minted but no route accepts them (killing the public API and the entire MCP server), OAuth login mints sessions that match no DB user, and chat attachments can be uploaded but never downloaded.
3. **Schema parity holes vs Jira/Linear.** No labels tables, no versions/releases, no components, no milestones, no board entity (WIP limits/swimlanes/column config), no issue rank/ordering, no resolution model, no workflow enforcement — each reported by 3–5 auditors independently.
4. **Operational fragility for self-hosters.** No CI exists, so nothing is verified automatically on push (the openapi drift suite itself passes at c519e8e — see Appendix A); the migration journal has non-monotonic timestamps that **silently skip migrations 0032/0044–0051 on upgrades**; `drizzle-kit generate` has been broken since migration 0012; a globally-unique issue-key index breaks the second org that picks a duplicate project key.
5. **Docs are systematically inverted from reality.** CLAUDE.md's gotchas ("AI is mocked", "coverage near-zero", "Hocuspocus not wired") are all false and actively mislead every AI-assisted session; `docs/STATUS.md` describes a Jan-2025 alpha; ROADMAP_2026 shows ~21 of 27 shipped items as still planned.

A large amount of _finished_ work is orphaned (analytics bento + chart suite, time-tracking panel, SSO/import/intake settings pages, quick-actions toolbar, `@tasknebula/types`, `resource-management.ts`, auth guard wrappers) — meaning several "missing features" are actually 1-day mounting/wiring tasks.

## 2. Top 15 Critical/High Gaps (ranked by user impact)

### 1. Cross-tenant authorization holes across ~20 API surfaces (read **and write**)

**Severity:** Critical · **Reported by:** api-issues, api-projects-sprints, api-admin-permissions, api-ai, api-search-notifications, web-kanban-board, web-backlog-roadmap, web-dashboard-analytics, web-settings-admin, web-docs-knowledge, security-posture, realtime-collab
**Impact:** Any authenticated user from any org can: rewrite another tenant's workflows and transitions, attach/detach any project's permission/security scheme, list any project's members (emails + full 50-flag permission matrix), pull any org's issues into any sprint, create/delete issue links on any issue, read any issue's activity history, full-text search any org (`/api/search/hybrid` trusts body `organizationId`), read any org's AI config, query any project's burndown/velocity/health, watch any issue, and receive **every tenant's realtime events** via the global SSE bus.
**Evidence:** `apps/web/src/app/api/workflows/route.ts` (+`[workflowId]`, statuses, transitions); `api/projects/[projectId]/{members,permission-scheme,security-scheme,workflow-statuses,workflow-transitions}/route.ts`; `api/sprints/route.ts` + `api/sprints/[sprintId]/issues/route.ts`; `api/permission-schemes/**`, `api/security-schemes/**`; `api/issues/[issueId]/{links,activities,time-in-status}/route.ts`; `api/watchers/route.ts`; `api/search/hybrid/route.ts`; `api/saved-filters/route.ts`; `api/push-subscriptions/route.ts`; `api/ai/capability/route.ts`, `api/ai/trace/[id]/route.ts`; `api/analytics/{burndown,velocity,project-health,insight}/route.ts`; `api/events/stream/route.ts`; `api/docs/spaces/route.ts` (project branch); `apps/web/src/lib/projects/server.ts` (`resolveProjectByIdOrKey` resolves keys globally across orgs).
**Fix:** Adopt the existing-but-unused canonical guards (`apps/web/src/lib/auth/access-control.ts`, `lib/auth/guards.ts`) as a single chokepoint; add org-membership checks to every route above; scope `resolveProjectByIdOrKey`/`resolveProjectId` key lookups to the caller's orgs; filter SSE events by subscriber org (payload already carries `organizationId`); add a 401/403/cross-org-404 guard-matrix test suite.

### 2. Postgres RLS documented as mandatory but completely absent; 60/107 tables lack `organization_id`

**Severity:** Critical · **Reported by:** db-rls-migrations, db-core-work-items, db-agile-planning, db-platform-tenancy, security-posture, tests-coverage, realtime-collab, web-docs-knowledge, web-chat-calls, api-search-notifications
**Impact:** No DB-level backstop — one forgotten `WHERE` (see Gap #1, where ~20 already exist) is a cross-tenant breach. Enterprise/trust docs overclaim isolation, which is worse than absence in a security review.
**Evidence:** zero `ENABLE ROW LEVEL SECURITY`/`CREATE POLICY` in `packages/db/drizzle/` (54 files); no `set_config`/session-GUC plumbing in `packages/db/src/client.ts`; tables without `organization_id` incl. `sso_configs`, `scim_tokens`, `content_embeddings`, `time_entries`, `sprints`, `notifications`, `watchers`, `issue_comments`, `attachments`, `issue_status_history`, `workflow_statuses/transitions`, initiatives child tables; claims in `CLAUDE.md`, `.claude/rules/database.md`, `docs/ARCHITECTURE.md`, `docs/FEATURES.md`.
**Fix:** Phase 1: backfill `organization_id` onto the ~25 genuinely tenant-scoped tables (via joins) + indexes. Phase 2: introduce a session GUC (`app.current_org_id`) set per-request, land RLS policies table-by-table starting with `issues`/`projects`/`documents`. Interim: correct the docs to "planned, not implemented."

### 3. Search is broken end-to-end (Cmd+K 405s, JQL route 500s, vector leg permanently empty)

**Severity:** Critical · **Reported by:** api-search-notifications, web-command-palette-ux, api-ai, performance
**Impact:** The flagship surface of a "keyboard-first" product doesn't work: every palette query fails with 405; the fallback `/api/search` would 500 anyway (references non-existent `issues.status` column, masked by the `: any` table typing, plus an invalid jsonb `LIKE` on labels); the embedding queue grows unbounded with **zero** workers, so hybrid/semantic search, Ask retrieval, AI estimates, and duplicate detection all silently return nothing. Even working results would 404 (`/work-items/${id}` vs real `/issues/[issueId]`).
**Evidence:** `apps/web/src/lib/command/use-omnibar-search.ts` (GET vs POST-only `api/search/hybrid/route.ts`; fallback only on 404/501; wrong hrefs); `api/search/route.ts:140,185,209`; `lib/search/embeddings.ts` (`drainEmbeddingQueue` — zero callers); `packages/db/drizzle/0035_hybrid_search.sql` (triggers enqueue forever).
**Fix:** POST the hybrid endpoint (or treat 405 as fallback); fix `/api/search` with a `workflow_statuses` join and jsonb `@>` containment; invoke `drainEmbeddingQueue()` from the existing janitor cron + enqueue on issue/comment writes; correct result hrefs to `/issues/${id}`; add org guard per Gap #1.

### 4. Core `issues` schema integrity: globally-unique key index breaks multi-tenancy; `: any` erases all type safety

**Severity:** Critical · **Reported by:** db-core-work-items, db-agile-planning, pkg-types-consistency, api-search-notifications
**Impact:** (a) `issue_key_idx` is unique on `key` alone while project keys are only unique per-org — two orgs picking project key "TASK" means the second org's **first issue insert fails** with a unique violation. (b) `export const issues: any` (also `issueComments`, `initiatives`) disables Drizzle inference across all 245 routes — the direct cause of the shipped `issues.status` runtime bug (Gap #3) and ~149 `as any` casts.
**Evidence:** `packages/db/src/schema/issues.ts:50,108,127`; `initiatives.ts:38`; `drizzle/0000_dear_puck.sql:580`; `packages/db/src/queries/issues.ts`.
**Fix:** One migration: `uniqueIndex` on `(organization_id, key)`. One refactor: type self-reference callbacks as `AnyPgColumn` (Drizzle's documented pattern, already approximated in `documents.ts`/`chat.ts`) and remove the three `: any` casts; then export `$inferSelect` model types from `packages/db`.

### 5. Migration journal silently skips migrations 0032/0044–0051 on upgraded installs

**Severity:** Critical · **Reported by:** db-rls-migrations
**Impact:** Any self-hosted DB that migrated at/after commit 9f64a44 and then upgrades **silently skips** SSO/SCIM tables, issue_status_history, standups, content_embeddings, the HNSW index, audit sinks, and import jobs — the exact drift class the `0053` catch-all migration admits already happened. Runs on every Docker boot.
**Evidence:** `packages/db/drizzle/meta/_journal.json` (0043 `when=1779000000000` precedes 0044–0051 at `1778900000000`; 18 entries share one timestamp); drizzle-orm's `created_at < folderMillis` apply rule; `packages/db/src/utils/migration.ts`; `docker-entrypoint.sh`. Related: orphan `0003_add_attachments.sql` never in journal; snapshots frozen at 0012 so `pnpm db:generate` is broken; duplicate prefixes (0019×2, 0020×3, 0026×2).
**Fix:** Renumber `when` values strictly increasing; ship a one-time repair that re-runs the idempotent 0032/0044–0051 SQL; add a post-migrate schema-drift assertion (expected-tables check) so skips fail loudly; regenerate a fresh drizzle-kit snapshot baseline; delete the orphan 0003 file.

### 6. Permission schemes & issue security levels: unauthorized CRUD + zero runtime enforcement (security theater)

**Severity:** Critical · **Reported by:** api-admin-permissions, db-platform-tenancy, api-issues, api-projects-sprints, web-settings-admin, api-search-notifications
**Impact:** Admins configure Jira-style permission schemes and issue security levels in a full UI — and nothing enforces them: no query anywhere filters by `issues.securityLevelId`, and authorization uses only hard-coded role maps. Meanwhile any logged-in user can CRUD any org's schemes/levels (no authz beyond session) and reassign any project's scheme. False sense of security is worse than absence.
**Evidence:** `apps/web/src/app/api/permission-schemes/**`, `api/security-schemes/**` (session-only checks, no Zod, no audit); `packages/db/src/schema/{permission-schemes,issue-security}.ts` referenced by nothing in `lib/`; `issues.securityLevelId` (`issues.ts:84`) never filtered in `lib/auth/access-control.ts`, list/search/export.
**Fix:** Guard all scheme routes with `hasPermission(orgId,'org:settings')` resolving org from the **scheme row** (audit-log-sinks routes are the template); enforce `securityLevelId` in `canReadIssue` + list/search/export queries (membership tables already exist); emit audit logs on scheme mutations; until enforced, flag the admin UI as non-enforcing.

### 7. API keys are minted but never accepted — public API and entire MCP server non-functional

**Severity:** Critical · **Reported by:** api-integrations, pkg-mcp-server, security-posture, api-issues
**Impact:** Programmatic access — table stakes vs Jira/Linear token-first APIs — is decorative. Every MCP tool call 401s (both stdio and HTTP transports); on top of that several MCP tools would 400 on contract mismatches (wrong priority enums, `subtask` type not creatable, `link_pr` shape doesn't exist server-side), the package isn't published to npm despite `npx` instructions, and advertised OAuth endpoints don't exist.
**Evidence:** `apps/web/src/app/api/api-keys/route.ts` (sha256-hashed keys; no consumer anywhere — every route uses `await auth()` cookies); `packages/mcp-server/src/{client,auth,http}.ts`; `tools/{link-pr,create-subtask,create-issue}.ts` vs `api/issues/route.ts:124-129`; `src/server.ts` (templates registered as fixed URIs).
**Fix:** Build a `resolveApiKeyAuth` helper (hash incoming `X-API-Key`/Bearer, look up `api_keys`, resolve user+org, update `lastUsedAt`) and wire it into route auth; add a `scopes text[]` column (copy `scim_tokens` pattern); fix the MCP enum/field mismatches; publish to npm and correct the README.

### 8. OAuth sign-in is broken: no DB adapter, sessions match no user — but buttons always shown

**Severity:** Critical · **Reported by:** auth-sso (provider registration confirmed by dx-agent-config; end-to-end flow broken)
**Impact:** GitHub/Google login mints a JWT whose `user.id` is the provider profile id — no `users` row is ever created, so every org-scoped API returns 401/403. The signin UI renders the buttons unconditionally even when providers aren't configured. Adjacent auth gaps compound it: no brute-force protection on credentials login, irrevocable 30-day JWTs (SCIM deactivation doesn't end sessions), no 2FA/passkeys, email verification never enforced, inconsistent email casing splits identities.
**Evidence:** `apps/web/src/auth.ts` (no adapter), `src/auth.config.ts:85-93`, `src/components/auth/signin-form.tsx:140-228`; dead Auth.js tables in `packages/db/src/schema/users.ts`.
**Fix:** Add `@auth/drizzle-adapter` or a `signIn` callback that find-or-creates the `users` + `accounts` rows; render buttons conditionally from configured providers; set `session.maxAge` + re-check `users.status` in the jwt callback; rate-limit `authorize()` and signup with existing helpers; normalize emails to lowercase everywhere.

### 9. Notification pipeline is hollow: mentions never notify, watchers decorative, no in-app rows for core events, digests/push never sent

**Severity:** High · **Reported by:** api-search-notifications, api-issues, web-issue-detail, web-chat-calls, realtime-collab
**Impact:** The bread-and-butter Jira/Linear loop is missing: @mentions are parsed/stored (issues _and_ chat) but notify no one — the UI doesn't even send them to the API; watchers receive nothing on any event; assigned/commented/status-changed produce **email only** (fire-and-forget, no queue), so the bell and Smart Inbox stay empty for core events; `digestFrequency` prefs and digest templates exist with no sending job; web-push subscriptions are stored with no sender (no `web-push` dependency).
**Evidence:** `apps/web/src/components/issues/issue-activity.tsx:74-92` + `lib/hooks/use-comments.ts` (mentions dropped client-side); `api/issues/[issueId]/comments/route.ts` (mentions persisted, never notified); `lib/notifications/send-notification.ts` (email-only); `lib/chat/server.ts:894-1003`; `packages/db/src/schema/watchers.ts` (only consumer: my-issues filter); `api/push-subscriptions/route.ts`.
**Fix:** Pass mentions in the comment POST; insert in-app notification rows + `notifyIssueEvent('issue_mentioned')` for mentions (enum/templates/prefs/inbox chips all exist); fan out to watchers in the issue update/comment paths; insert in-app rows beside the existing email calls for assigned/commented/status-changed; add a digest cron + `web-push` sender.

### 10. Workflow transitions are stored but never enforced; `statusId` not validated at all

**Severity:** High · **Reported by:** db-agile-planning, api-issues, api-projects-sprints, web-kanban-board, web-settings-admin
**Impact:** The workflow editor (conditions/validators/post-functions JSONB, transition CRUD) is write-only data — any issue can move to any status; PATCH accepts a `statusId` from another org's workflow (the issue silently vanishes from every board column). The editor is also half-built: statuses can never be renamed/reordered/deleted (no `[statusId]` route exists), transitions can't be edited, and two divergent workflow UIs coexist.
**Evidence:** `apps/web/src/app/api/issues/[issueId]/route.ts` (~line 354, blind set); `api/issues/bulk/route.ts`; zero references to `workflowTransitions` under issue routes; `packages/db/src/schema/workflows.ts:42-51`; `projects.defaultWorkflowId` bare text, no FK (`projects.ts:33`).
**Fix:** On status change, validate `statusId` belongs to the project's workflow and that a `workflow_transitions` row permits the move (evaluate validators/post-actions or explicitly defer them); add `statuses/[statusId]` PATCH/DELETE with in-use 409; make `default_workflow_id` a real FK; consolidate the two editors.

### 11. AI last mile: flagship chat is a stub, cost guard bypassed by half the spend paths

**Severity:** High · **Reported by:** api-ai, web-command-palette-ux, docs-accuracy
**Impact:** Contrary to CLAUDE.md, the AI backends are real (draft, assist, triage, Ask RAG with citations, agents, digests) — but the AI Sidecar returns hardcoded "(stub)" text and nothing consumes the completed `/api/ask` SSE stream (the palette even sends a malformed payload that 400s silently). Meanwhile the org kill switch/budget doesn't actually stop spend: ask, triage, catch-me-up, insight, standup, janitor, and embeddings all bypass `runWithBudget`; prompt-injection wrapping and PII redaction helpers are built but never applied to prompts embedding other users' content; auto-triage-on-create is never invoked.
**Evidence:** `apps/web/src/components/ai/ai-sidecar-provider.tsx`; `components/command/command-palette.tsx` (`{prompt}` vs required `{query}`); `lib/ai/budget.ts` vs `app/api/ask/route.ts`, `lib/agents/{triage,standup,janitor}.ts`, `api/analytics/insight/route.ts`; `lib/ai/safety/{sandbox,redact}.ts` (zero production callers); `lib/agents/triage-enqueue.ts` (no caller).
**Fix:** Wire the sidecar to `/api/ask` SSE; fix the palette payload; wrap all remaining LLM paths in `runWithBudget`; apply `wrapUntrustedContent()` in issue-assist/triage/ask prompt builders; call `enqueueTriageOnCreate` from POST `/api/issues`; resolve workspace credentials in `/api/ask`.

### 12. Chat: attachments can never be downloaded; DB-configured LiveKit breaks calls entirely

**Severity:** Critical (within chat) · **Reported by:** web-chat-calls
**Impact:** Every chat file upload 404s for every user — the uploads route only authorizes issue/doc attachments, never chat JSONB attachments. Separately, deployments that configure LiveKit via the Admin UI (DB path) get 503 on call start and dead webhooks, because call-start and webhook verification read env vars only — calls never work.
**Evidence:** `apps/web/src/app/api/uploads/[filename]/route.ts:75-156` (`userCanAccessFile`); `api/conversations/[roomId]/messages/route.ts:92-124`; `lib/chat/server.ts:2056,2421` (env-only `getLivekitStatus`) vs `lib/chat/livekit.ts:77-97` (DB-first resolver); `api/chat/livekit/webhook/route.ts:6-15`.
**Fix:** Extend `userCanAccessFile` to resolve chat attachments via the room's project (or insert rows into `attachments` at upload); switch call-start/token/webhook to the DB-first `resolveLivekitConfigForServer`.

### 13. Upload & egress security: stored XSS via inline SVG, no MIME allowlist, SSRF-able webhooks, zero security headers

**Severity:** High · **Reported by:** security-posture, web-docs-knowledge, api-issues, web-chat-calls, api-integrations
**Impact:** Any org member can upload an SVG with `<script>` that executes same-origin for teammates (uploads served `inline` as `image/svg+xml`, no CSP/nosniff anywhere — `next.config.ts` sets no headers at all). Any member with webhook:create can probe internal networks/cloud metadata via outbound webhooks/test deliveries (plain `z.string().url()`, response bodies persisted and readable). Outbound SMTP also disables TLS cert verification.
**Evidence:** `apps/web/src/app/api/uploads/[filename]/route.ts`; `api/public/docs/[token]/assets/...`; `api/issues/[issueId]/attachments/route.ts` (size check only); `lib/webhooks/dispatcher.ts`; `lib/audit/sink-dispatcher.ts`; `apps/web/next.config.ts`; `lib/email/sender.ts` (`rejectUnauthorized: false`).
**Fix:** `headers()` block with CSP/nosniff/X-Frame-Options/HSTS; serve SVG/unknown types as `attachment`; MIME/extension allowlist on uploads; DNS-resolved private-IP blocklist + `redirect:'error'` in webhook/sink dispatchers; remove the TLS bypass (or gate behind explicit env flag).

### 14. No issue rank/ordering anywhere — manual board, backlog, and sprint ordering cannot persist

**Severity:** High · **Reported by:** db-core-work-items, db-agile-planning, api-projects-sprints, web-kanban-board, web-backlog-roadmap
**Impact:** Drag-to-reorder within a kanban column is an explicit no-op; backlog grooming order is impossible; everything renders `createdAt desc`. This is Jira LexoRank / Linear sortOrder table stakes, and the kanban UI is already wired with `SortableContext` for ordering that can never save.
**Evidence:** no rank/position column in `packages/db/src/schema/issues.ts`; `apps/web/src/components/kanban/kanban-board.tsx:135` (`if same status → return`); `api/issues/route.ts:258`; no reorder endpoint.
**Fix:** Add a `rank` column (LexoRank string or fractional numeric) + migration; persist drop index on drag (within + across columns); order list/board/backlog queries by rank; add a reorder endpoint used by backlog DnD.

### 15. Unpaginated core endpoints + no virtualization — collapses at Jira/Linear scale

**Severity:** High · **Reported by:** performance, api-issues, web-kanban-board, web-backlog-roadmap
**Impact:** `GET /api/issues`, `my-issues`, comments, activities, watchers return **every row**; the SSE sync hook invalidates broad keys so each issue event re-downloads the full set for every client; kanban/list render all rows with no windowing; `issues.organization_id` — the org-wide list/search filter — has **no index**. A 5–10k-issue org degrades badly.
**Evidence:** `apps/web/src/app/api/issues/route.ts:222-297` (no limit; `total = issuesData.length`); `api/issues/my-issues/route.ts:73-82`; `lib/hooks/use-realtime-sync.ts:78`; index block `packages/db/src/schema/issues.ts:108-122` (no org index); no react-window/virtuoso/tanstack-virtual in `apps/web`.
**Fix:** Capped `limit/offset` (or cursors) on issues/comments/activities mirroring `/api/search`; indexes on `issues(organization_id[, created_at])`, `issues(epic_id)`, `attachments(file_path)`, `attachments(issue_id)`; virtualize board columns and lists; switch invalidation to targeted cache updates.
