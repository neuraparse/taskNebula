# Project Management Tools - Competitive Analysis 2025

## Research Date: November 2025

This document contains the latest features and capabilities of leading project management tools as of November 2025, compared against TaskNebula.

## 1. JIRA (2025 Latest Features)

### AI-Powered Features
- **AI Work Breakdown**: Automatically breaks down large projects into individual tasks with AI-recommended subtasks
- **AI Work Creation**: Create work items directly from Confluence pages or Slack messages using AI
- **Loom AI Workflows**: Convert Loom recordings into work items with AI-generated descriptions
- **Suggested Topics** (JSM): AI identifies gaps in knowledge base based on customer requests
- **Draft Reply** (JSM): AI helps create draft messages for help seekers

### Customization & Templates
- **Custom Project Templates**: Create reusable templates with pre-configured workflows, automations, fields, and permissions
- **Form Templates**: Pre-built templates for frequent requests (website updates, content creation)

### Enhanced Workflows & Visualization
- **Conditional Formatting**: Create rules to highlight important rows and cells in lists
- **Attachments Hub**: Centralized management for all project images and attachments
- **New Workflow Editor**: Enhanced configuration for function validators

### Platform Improvements
- **Forge Automation Actions**: Extend automation platform with Forge-based custom actions
- **Refreshed UI**: New navigation UI with updated Atlassian Design System (97% retention rate)
- **SCIM 2.0 Customer Provisioning**: Automated account management for JSM

## CONCLUSION

TaskNebula has solid foundational features comparable to 2024-era tools, but is missing the **AI-first capabilities** that define 2025 project management platforms. To remain competitive:

1. **Immediate Focus**: AI agents, semantic search, template system
2. **3-Month Goal**: GitHub/Slack integrations, advanced automation
3. **6-Month Goal**: Portfolio management, resource planning, mobile apps

The competitive landscape has shifted from "workflow management" to **"AI-powered autonomous work execution"** - this is the critical gap to address.
